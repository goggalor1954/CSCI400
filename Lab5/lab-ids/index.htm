Intrusion Detection System

An Intrusion Detection System (IDS) is a device or software application that monitors network and/or system activities for malicious activities or policy violations and produces reports to a Management Station. -- Wikipedia

Types of IDS
  • Network intrusion detection system (NIDS)
  • Host-based intrusion detection system (HIDS)
  • Perimeter Intrusion Detection System (PIDS)

Snort

You can get Snort from its homepage, and get the user manual from here. In order to use Snort in Windows, you may need to install WinPcap.

Snort has four running modes: Sniffer mode, Packet Logger mode, Network Intrusion Detection System mode, Inline mode. Now we just focus on Network Intrusion Detection System(NIDS) mode. To use NIDS mode, you need a rule file. You can modify the rule file in ./snort/etc. You also can download the rule package from here. More details please see the user manual section 1.4.

Bro Intrusion Detection System

You can get Bro from its homepage. You can find user manuals in here. Bro works with Linux and Solaris as well, but the performance is best under FreeBSD. Also, you need install tcpdump and libpcap. You can download them from here.

More details please see the online user manual.

Setting up an IMAP server on Ubuntu


  1. Install the packages

    sudo apt-get install dovecot-imapd dovecot-pop3d

  2. Configure the protocol you need to be used by appending the protocol in the file /etc/dovecot/dovecot.conf:

    protocols = pop3 pop3s imap imaps

  3. Choose the mailbox format you would like to use. Dovecot supports maildir and mbox formats. Choose mbox for simplicity. Edit the file /etc/dovecot/dovecot.conf and change the line

    mail_location = mbox:~/mail:INBOX=/var/spool/mail/%u # (for mbox)

  4. Restart the service

    sudo /etc/init.d/dovecot restart

  5. Use telnet to check that dovecot is working properly.

    telnet localhost imap
